Hands
Due to the complexity I'd rather to have ruleset hard-coded instead of reading from external file. 
Other Fan related information are in fan.csv. Oh it's incomplete yet.
A separate pyfile is needed to handle those.
Output language is Traditional Chinese by default, so column "name_c" and "desc_c" matters.
See fan_explanation_1.txt for some rules and implement those Fan first. 
Usually it's checked from the Fan with highest value all the way down to lowest value.
Say Fan A (600 pts), B(200 pts), C (100 pts), and A's condition is met.
If "Override B" is mentioned in A's rule, then checking for B will be skipped.
Remember a same fan can only score once.

Scoring
Case 1: Excellence Fan achieved.
All achieved Fans with a value <500 are ignored.
Achieved Fan with the most value will be the main one: applied normally.
All other achieved Fans >=500 pts: applied with halved score (round down to nearest 100, but no less than 300)
If there's multiple Fan with the most value, use the one with lower ID as main one. Example:
[ALL HONORS] (109) 3000 pts, [MAJOR THREE DRAGONS] (110) 3000 pts, Three Quads 2000pts, Win after a Quad 500 pts:
[ALL HONORS] is the main one. 3000 pts. 
[MAJOR THREE DRAGONS] +1500 pts, Three Quads +1000 pts, Win after a Quad +300 pts
Total value 4800 pts.

Case 2: No Excellence Fan achieved.
Add up all applicable Fans, unless a Fan with higher value mentioned to ignore some Fan with lower value, then apply the rule of the Fan with higher value.
If sum is no more than 3000 points then that's the total points.
If sum is more than 3000 points, the exceeding part is halved, round down to nearest 100.
[3000~3200)=3000, [3200~3400)=3100, [3400~3600)=3200 and so on.
